# 実装ロードマップ

**バージョン**: 1.0
**作成日**: 2026年6月
**スコープ**: MVP の実装順序・依存関係・段階移行

> 機能 ID は [02 機能設計](02_function_design.html) に対応。

---

## 1. フェーズ全体像

```
[P0 基盤] → [P1 記録の核] → [P2 段取り] → [P3 報酬] → [P4 チーム] → [P5 仕上げ]
                                                                  → [2nd 以降]
```

MVP は P0〜P5。GPS / 全国ランキング / 育成マッチョ / 外部連携は 2nd 以降。

## 2. スプリント計画

| 項目 | 内容 | 対応 |
|---|---|---|
| リポジトリ/モノレポ | mobile(Expo)+ api(Hono)+ 共有 zod スキーマ | — |

### P4 — チーム

## 3. 依存グラフ

```
BASE-01 認証
   └─ BASE-02 プロフィール
         ├─ F03 記録 ──────────────┐
         │     └─ F04 報酬 ── F05 ペナルティ
         ├─ F01 スケジュール ── 06 通知 ── F02 チェックリスト
         └─ F06 チーム(F03 の記録データに依存)
```

- **F03(記録)が中心依存**。報酬・チーム・ランキングはすべて記録データに依存する
- 通知(06)は F01/F02 の後

---

## 4. 段階移行(MVP → 2nd)

| 2nd 機能 | 前提・移行時の論点 |
|---|---|
| GPS チェックイン | 位置許可・判定精度。F03 の「証明」を自己申告から拡張 |
| 全国ランキング「日本で○位」 | 一定のユーザー母数。集計バッチの追加 |
| 育成マッチョ | 記録に連動するキャラ育成。記録データを流用 |
| LINE / カレンダー連携 | OAuth / Messaging API。通知経路の追加(06 を拡張) |

> 2nd の各機能は、それぞれ独立した spec → plan → 実装サイクルで進める。

---

## 5. 開発規律

- 設計書を SSoT とし、**実装が設計と食い違ったらコードを直す前に設計書を直す**
- 未確定事項([02](02_function_design.html) §10 / [05](05_tech_stack.html) §3 / [06](06_notification_design.html) §8)は、着手前に確定して該当節を更新する
- `main` に直 push しない / git hook を skip しない
