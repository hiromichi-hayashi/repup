画面設計書

バージョン: 1.0 作成日: 2026年6月 スコープ: MVP の画面一覧(SC-01〜SC-11)と遷移

機能との対応は 02 機能設計。React Native + Expo / iOS 優先。

1. 画面一覧

ID 画面 対応機能 概要
SC-01 ホーム F01/F03/F04 今日の予定・連続カウント・記録ボタン
SC-02 オンボーディング BASE-02 目的・頻度・時間帯・通知許可
SC-03 サインイン BASE-01 Apple / メール
SC-04 スケジュール一覧/編集 F01 行く日の設定・繰り返し
SC-05 準備チェックリスト F02 予定ごとの項目チェック
SC-06 チェックリストテンプレート編集 F02 項目のカスタム
SC-07 記録入力 F03 記録する/行かなかった の確定
SC-08 成長・トロフィー F04 連続・累積・ランク・グラフ
SC-09 チーム F06 メンバー進捗・参加/作成
SC-10 チーム内ランキング F06 順位表示
SC-11 設定 BASE-02 プロフィール・通知設定

2. ナビゲーション構造

下タブ(Bottom Tab)4 つを基本に、モーダルで詳細を出す。

[Tab] ホーム(SC-01) ─ 記録入力(SC-07, モーダル)
[Tab] スケジュール(SC-04) ─ チェックリスト(SC-05) / テンプレ編集(SC-06)
[Tab] チーム(SC-09) ─ ランキング(SC-10)
[Tab] 成長(SC-08)
        設定(SC-11) は各所からアクセス

初回のみ: サインイン(SC-03) → オンボーディング(SC-02) → ホーム

3. 画面詳細

SC-01 ホーム

目的: 「いかなきゃ」と「続いている実感」を一目で。アプリの主役画面。

  • 今日の予定(あれば時間帯)。なければ「予定を追加」導線
  • 連続カウントを大きく表示(「12回連続いけてる!💪」)
  • 現在のトロフィーランク(ブロンズ/シルバー/ゴールド)
  • 主要 CTA: 記録する(→ SC-07)
  • 過去のブランク日があれば「確定する」導線(→ SC-07)

SC-02 オンボーディング

目的: 動機づけの初期設定。

  • 目的の入力/選択 → 週の目標回数 → 既定時間帯(朝を推奨表示)→ 通知許可
  • 完了で POST /onboarding/complete、ホームへ

SC-03 サインイン

  • Sign in with Apple ボタン / メール+パスワードでサインアップ・サインイン
  • パスワードを忘れた場合のリセット導線
  • 初回は User 作成 → SC-02 へ

SC-04 スケジュール一覧 / 編集

目的: 行く日を決める(F01)。

  • カレンダー or リスト表示。単発追加 / 繰り返し(曜日選択)
  • 各予定に時間帯。朝をデフォルトに寄せる UI
  • 予定からチェックリスト(SC-05)へ

SC-05 準備チェックリスト

  • 選択した予定の項目を ON/OFF
  • 未達があれば前日アラートが飛ぶ旨を明示(§06 連動)
  • 「項目を編集」→ SC-06

SC-06 チェックリストテンプレート編集

  • 項目の追加/削除/並べ替え(例: コワーキング予約済み?)
  • PUT /checklist-templates

SC-07 記録入力(モーダル)

目的: 「行った証明」の確定(F03)。

  • 記録する: メモ/タグ/時間(任意)→ POST /records(RECORDED)
  • 行かなかった: POST /records/skip(SKIPPED)
  • 対象日が選べる(当日 / 過去のブランク日)
  • 状態遷移は 02 §F03 に準拠

SC-08 成長・トロフィー

  • 連続カウント・最長連続・累積回数
  • 現在ランクと次ランクまでの条件
  • 記録推移グラフ(週次/月次)
  • 降格が起きた場合の表示(F05)

SC-09 チーム

  • 所属チームのメンバー進捗(連続・直近記録)
  • チーム作成 / 招待コードで参加
  • ランキングへ(SC-10)

SC-10 チーム内ランキング

  • 指標(連続 or 期間内記録数)で順位表示
  • 自分の順位をハイライト
  • 表示はチーム内順位のみ(対象外は 01 §7.3)

SC-11 設定

  • プロフィール(表示名・目的・目標頻度・既定時間帯)
  • 通知設定(リマインド時刻・前日アラート ON/OFF)
  • サインアウト

4. 状態・空表示

画面 空/異常時
SC-01 予定なし → 追加導線。記録ゼロ → 「最初の1回を記録しよう」
SC-09 未所属 → 作成/参加の二択を大きく
SC-10 メンバー1人 → 「仲間を招待しよう」
全般 オフライン → 再試行。記録送信失敗 → 再送導線(05 のリトライ方針)

5. デザイン方針(MVP)

  • iOS のネイティブ感を尊重しつつ、連続カウント/トロフィーは達成感が出るビジュアルに
  • 朝の行動を促す配色・コピー(明るい・前向き)
  • 詳細なビジュアルデザイン(配色トークン・コンポーネント)は実装フェーズで Figma 等にて確定
