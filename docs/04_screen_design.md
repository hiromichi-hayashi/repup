## 2. ナビゲーション構造

下タブ(Bottom Tab)4 つを基本に、モーダルで詳細を出す。

```
[Tab] ホーム(SC-01) ─ 記録入力(SC-05, モーダル)
[Tab] スケジュール(SC-04)
[Tab] チーム(SC-07) ─ ランキング(SC-08)
```

## 3. 画面詳細

### SC-01 ホーム

**目的**: 「いかなきゃ」と「続いている実感」を一目で。アプリの主役画面。

- 今日の予定(あれば時間帯)。なければ「予定を追加」導線
- **連続カウント**を大きく表示(「12回連続いけてる!💪」)
- 現在のトロフィーランク(6段階: ブロンズ→…→マスター)
- 主要 CTA: **記録する**(→ SC-05)
- 過去のブランク日があれば「確定する」導線(→ SC-05)

### SC-02 初回設定(設定の目標設定を流用)

**目的**: 初回の目標設定。**専用 UI は作らず、設定(SC-09)の「目標設定」フォームをそのまま初回に通す**([02](02_function_design.html) §3.3)。

- 内容は設定の目標設定と同じ: 目的 → **通う日(繰り返しスケジュール)**(毎週の曜日 / N週ごと / 毎月の日) → 既定時間帯(**朝を推奨表示**)
- 初回のみ **通知許可** を要求
- 完了でアカウント設定を保存(`PATCH /account`、`onboardedAt` 記録)、通う日を作成(`POST /schedules`)し、ホームへ
- 通う日は連続カウント(F04)の基準。以後の編集は設定(SC-09)で行う

### SC-03 サインイン

- Sign in with Apple ボタン / メール+パスワードでサインアップ・サインイン
- パスワードを忘れた場合のリセット導線
- 初回は User 作成 → SC-02 へ

### SC-04 スケジュール一覧 / 編集

**目的**: 通う日を決める(F01)。連続カウント(F04)の基準にもなる。

- カレンダー or リスト表示。繰り返しパターンを選んで設定:
  - **毎週**: 曜日選択(月・水・金 など)
  - **N週ごと**: 曜日 + 間隔(2週に1回 など)
  - **毎月**: 日付選択(毎月1日・15日 など)
  - **単発**: 特定日
- 各予定に時間帯。**朝を既定**に寄せる UI

### SC-05 記録入力(モーダル)

**目的**: 「行った証明」の確定(F03)。

- **記録する**: メモ/タグ/時間(任意)→ `POST /records`(RECORDED)
- **行かなかった**: `POST /records/skip`(SKIPPED)
- 対象日が選べる(当日 / 過去のブランク日)
- 状態遷移は [02](02_function_design.html) §F03 に準拠

### SC-06 成長・トロフィー

- 連続カウント・シーズン内最高連続・最長連続・累積回数
- **シーズン制ランク**(6段階): 現在ランクと**次ランクまでの連続**、シーズン残り期間
- 過去シーズンの最高ランク履歴
- 記録推移グラフ(週次/月次)

### SC-07 チーム

- **チーム目標**: スローガン(例「朝ジム部」)と**共通の通う日**を表示
- 共通の通う日に「**自分の予定に取り込む**」導線(`POST /teams/:id/adopt-schedule`)
- 所属チームのメンバー進捗(連続・直近記録)
- チーム作成 / 招待コードで参加
- オーナーはチーム目標(スローガン・推奨通う日)を編集(`PATCH /teams/:id`)
- ランキングへ(SC-08)

### SC-08 チーム内ランキング

- 指標(連続 or 期間内記録数)で順位表示
- 自分の順位をハイライト
- 表示は**チーム内順位のみ**(対象外は [01](01_concept.html) §7.3)

### SC-09 設定

- アカウント情報(表示名)/ 目標設定(目的・週回数・既定時間帯)
- 通知設定(リマインド時刻・前日アラート ON/OFF)
- サインアウト

## 4. 状態・空表示

| 画面 | 空/異常時 |
|---|---|
| SC-01 | 予定なし → 追加導線。記録ゼロ → 「最初の1回を記録しよう」 |
| SC-07 | 未所属 → 作成/参加の二択を大きく |
| SC-08 | メンバー1人 → 「仲間を招待しよう」 |
| 全般 | オフライン → 再試行。記録送信失敗 → 再送導線([05](05_tech_stack.html) のリトライ方針) |

## 5. デザイン方針

- iOS のネイティブ感を尊重しつつ、連続カウント/トロフィーは**達成感が出る**ビジュアルに
- 朝の行動を促す配色・コピー(明るい・前向き)
- 詳細なビジュアルデザイン(配色トークン・コンポーネント)は実装フェーズで Figma 等にて確定
